BEM Naming convention

Block:

A standalone component or module that has a meaningful purpose and can be reused across the website. Block names should be descriptive and represent the main purpose of the component.

Example: .button { }

Element:

Parts of a block that have no meaning or purpose on their own and are semantically tied to the block. Element names are separated from the block name by a double underscore __.

Example: .button__text
.navbar__link
.card__image { }

Modifier:

Variants or states of a block or element that alter its appearance, behavior, or state. Modifier names are separated from the block or element name by a double hyphen --.

Example: .button--primary { }
.navbar__link--active { }
.card__image--rounded { }
